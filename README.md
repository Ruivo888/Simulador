Esta ferramenta interativa permite a visualização de fenômenos que são, por natureza, invisíveis, 
facilitando a compreensão da Conversão Eletromecânica de Energia e do comportamento de circuitos magnéticos em regime variável.

O código é uma aplicação web de página única (SPA) que utiliza a tríade HTML5, CSS3 e JavaScript, com foco no elemento Canvas para renderização gráfica de alta performance.1. Estrutura de Interface (HTML & CSS)O esqueleto do site organiza os elementos para que a experiência do usuário seja intuitiva:Header: Identifica o autor e a instituição, estabelecendo o contexto acadêmico.Painel de Instruções: Uma seção lateral (<div class="instructions">) que explica as variáveis físicas para quem não é da área.Área de Simulação: Contém o <canvas>, que é o "quadro negro" onde os desenhos são feitos via código, e os controles deslizantes (<input type="range">) para manipulação de dados em tempo real.CSS Moderno: Utiliza Flexbox e Grid para garantir que o simulador fique centralizado e organizado, com uma estética Dark Mode que remete a instrumentos de laboratório modernos.2. O Motor de Cálculos (JavaScript)O comportamento físico é simulado dentro de um objeto chamado params, que armazena os valores atuais de $i_1, N_1, N_2$ e $\mu$.A Lógica de Fluxo e InduçãoDiferente de um vídeo estático, o código recalcula os valores a cada quadro (aproximadamente 60 vezes por segundo):Fluxo Unitário ($\Phi$): Calculado como uma função direta da corrente e da permeabilidade.Fluxo Concatenado ($\lambda$): Multiplica o fluxo pelo número de espiras ($N \cdot \Phi$).Diferencial de Faraday: O código armazena o valor do fluxo do quadro anterior (prevLambda). A diferença entre o fluxo atual e o anterior dividida pelo tempo é o que gera a FEM Induzida.Nota: É por isso que, se você parar de mexer nos controles, a FEM volta para zero, simulando fielmente que apenas a variação do fluxo gera tensão.3. Renderização Gráfica (O Canvas)Esta é a parte visualmente mais rica do projeto, onde três funções principais atuam:drawCoil() (As Bobinas)Utiliza equações matemáticas de elipses para desenhar as espiras. O número de ciclos do laço for é controlado pelo slider $N$, permitindo que o usuário veja a bobina "crescer" ou "encolher" fisicamente na tela.drawFluxLines() (O Campo Magnético)Esta função desenha as linhas de campo usando Curvas de Bézier.Animação Dinâmica: O código utiliza lineDashOffset associado ao tempo (Date.now()). Isso cria o efeito das linhas "caminhando" pelo espaço.Sentido do Fluxo: Se a corrente é invertida, o cálculo do offset torna-se negativo, fazendo com que as linhas de fluxo mudem de direção visualmente.drawArrow() (Vetor Direcional)Desenha uma seta vetorial que aponta para onde o fluxo está seguindo, facilitando a interpretação da polaridade magnética.4. O Ciclo de Animação (requestAnimationFrame)O coração do simulador é a função animate(). Ela funciona em um loop infinito:Limpa a tela (clearRect).Lê os valores atuais dos sliders.Realiza os cálculos de física.Desenha os novos quadros com as posições atualizadas.Atualiza o texto do painel de estatísticas com os resultados de $\lambda$ e $\epsilon$.Este método garante que a transição entre os estados seja fluida (60 FPS), proporcionando uma ferramenta de aprendizado visual muito mais poderosa que um gráfico estático.
